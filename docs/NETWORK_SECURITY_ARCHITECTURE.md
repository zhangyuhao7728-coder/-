## 二、安全设计原则

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                          🔐 安全设计三大原则                                 │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  1️⃣ Zero Trust (零信任)                                                  │
│     ┌─────────────────────────────────────────────────────────────┐        │
│     │ • 从不信任，始终验证                                           │        │
│     │ • 每次请求都验证身份和权限                                     │        │
│     │ • 最小化权限原则                                              │        │
│     └─────────────────────────────────────────────────────────────┘        │
│                                                                             │
│  2️⃣ Least Privilege (最小权限)                                           │
│     ┌─────────────────────────────────────────────────────────────┐        │
│     │ • 只授予必需的最小权限                                         │        │
│     │ • 文件系统隔离 (禁止访问 ~/.ssh, ~/.aws)                      │        │
│     │ • 命令白名单 (只允许特定命令)                                  │        │
│     │ • 网络隔离 (仅本地访问)                                        │        │
│     └─────────────────────────────────────────────────────────────┘        │
│                                                                             │
│  3️⃣ Defense in Depth (纵深防御)                                           │
│     ┌─────────────────────────────────────────────────────────────┐        │
│     │ • 多层安全防护                                                │        │
│     │ • 每层独立防御                                                │        │
│     │ • 单层失效不影响整体                                          │        │
│     └─────────────────────────────────────────────────────────────┘        │
│                                                                             │
│  4️⃣ Audit Everything (审计一切)                                            │
│     ┌─────────────────────────────────────────────────────────────┐        │
│     │ • 记录所有操作                                                │        │
│     │ • 可追溯性                                                    │        │
│     │ • 异常检测                                                    │        │
│     └─────────────────────────────────────────────────────────────┘        │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 四、安全模块详解

### 4.1 Telegram Guard (用户白名单)

```
位置: openclaw_modules/telegram_guard.py

功能:
• 验证 Telegram 用户身份
• 只允许白名单用户
• 记录用户操作

实现:
ALLOWED_USERS = {8793442405}  # 你的ID
```

### 4.2 Prompt Guard (Prompt防御)

```
位置: security/prompt_guard.py

检测模式:
• 忽略指令: "ignore previous instructions"
• 越狱攻击: "jailbroken", "DAN mode"
• 敏感文件: "read /etc/passwd"
• 社会工程: "pretend to be"
```

### 4.3 Command Guard (命令白名单)

```
位置: security/command_guard.py

允许命令:
• python, python3, pip
• projects/crawler/**
• scripts/**
• git status/log/diff (只读)

禁止:
• rm -rf /
• curl | sh
• sudo
```

### 4.4 FileSystem Guard (文件隔离)

```
位置: security/filesystem_guard.py

允许:
• ~/项目/Ai学习系统
• ~/项目
• /tmp
• ~/Downloads

禁止:
• ~/.ssh
• ~/.aws
• /etc
• /System
```

### 4.5 Network Guard (网络控制)

```
功能:
• IP 白名单
• 端口控制
• 出站流量限制
• DNS 安全
```

### 4.6 Token Guard (令牌验证)

```
功能:
• JWT 验证
• Token 轮换
• 过期检查
• 签名验证
```

### 4.7 Sandbox Guard (沙盒执行)

位置: security/sandbox.py

功能:

### 4.8 Audit Logger (审计日志)
