## 什么是 RAG？

**RAG = 检索增强生成**

简单说：把你自己的资料（PDF、文档、笔记）喂给 AI，让它能基于你的资料回答问题。

**对比：**
| 方式 | 回答来源 | 缺点 |
|------|---------|------|
| 普通 AI | 训练数据 | 不知道你的私人资料 |
| RAG | 你的文档 | 知道！ |

---

## 🔬 RAG 底层原理

### 第一层：为什么需要 RAG？

**AI 的本质是一个"语言模型"——它只会"预测下一个词"**

```
输入: "今天天气真"
模型输出: "好" (因为训练数据里"今天天气真"后跟"好"的概率最高)
```

**问题：** 模型的知识来自训练数据，有两个致命缺陷：

| 缺陷 | 后果 |
|------|------|
| 知识截止 | 它不知道 2024 年后的事 |
| 无私有知识 | 它不知道你的笔记、文档、公司内部资料 |

**RAG 的核心思想：** 不要让 AI 凭空编答案，而是先把相关资料找出来，再让 AI 基于资料回答。

---

### 第二层：向量——RAG 的灵魂

**文字是怎么"存储"在计算机里的？**

传统存储：`"苹果" -> 字符串`

RAG 存储：`"苹果" -> [0.12, -0.34, 0.78, 0.02, ...]` (一个向量)

```
苹果     -> [0.12, -0.34, 0.78, ...]
iPhone   -> [0.11, -0.33, 0.79, ...]  ← 相似！
香蕉     -> [0.08, -0.40, 0.21, ...]  ← 较远
汽车     -> [-0.55, 0.22, 0.01, ...]  ← 远离
```

**这就是向量 Embedding 的核心：语义相似的文字，向量也"靠近"。**

---

### 第三层：向量数据库——高速检索

**传统数据库查询：**
```sql
SELECT * FROM docs WHERE content LIKE '%苹果%'
```
→ 只能精确匹配，不能语义搜索

**向量数据库查询：**
```python
# 找和"水果"最像的文档
results = vector_db.search(query_vector="水果的向量", top_k=5)
```
→ 返回语义相关的文档，不管关键词是否一样

---

### 第四层：完整 RAG 流程

```
┌─────────────────────────────────────────────────────────────────┐
│                      用户提问:                                   │
│                 "我的笔记里关于Python爬虫的内容是什么？"          │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  1. Query Embedding (查询向量化)                                │
│     "我的笔记里关于Python爬虫的内容是什么？"                      │
│     -> [0.23, -0.11, 0.56, ...]                                 │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  2. Vector Search (向量检索)                                    │
│     在向量数据库中找最相似的 Top-K 条文档                        │
│     找到: ["Python爬虫教程.txt", "BeautifulSoup用法.md", ...]   │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  3. Context Assembly (上下文组装)                               │
│     把检索到的内容组装成提示词:                                  │
│     """                                                         │
│     基于以下资料回答用户问题:                                    │
│                                                             │
│     资料1: Python爬虫教程.txt                                   │
│     使用requests + BeautifulSoup爬取网页...                    │
│                                                             │
│     资料2: BeautifulSoup用法.md                                 │
│     find()和select()方法的区别...                              │
│                                                             │
│     用户问题: 我的笔记里关于Python爬虫的内容是什么？              │
│     """                                                         │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  4. LLM Generation (LLM生成答案)                                │
│     把组装好的提示词发给 AI，返回答案                            │
└─────────────────────────────────────────────────────────────────┘
```

---

### 第五层：关键技术细节

| 组件 | 主流选择 | 作用 |
|------|-------- **Embedding 模型**-|------|
| | sentence-transformers | 把文字转成向量 |
| **向量数据库** | Chroma / FAISS / Milvus | 存储 + 高速检索 |
| **文档加载器** | LangChain / LlamaIndex | PDF/TXT/MD 解析 |
| **LLM** | Ollama / OpenAI API | 生成答案 |

## 🤖 OpenClaw 的 Agent 转发机制

### 1. 主动 Spawn（生成子 Agent）

```python
# Spawn 一个独立子 Agent 帮你干活
sessions_spawn(
    runtime="subagent",  # 或 "acp"
    task="帮我爬取某网站数据并整理成CSV",
    mode="run"  # run=一次性的 | session=持续对话
)
```

**适用场景：** 并行处理多个任务，每个子 Agent 独立完成

---

### 2. 消息转发（跨会话）

```python
# 把任务发给另一个 Agent 会话
sessions_send(
    sessionKey="xxx",
    message="帮我完成这部分工作"
)
```

**适用场景：** 把任务转给专门处理某类问题的 Agent

---

### 3. 子 Agent 管理

```python
# 查看当前有哪些子 Agent 在跑
subagents(action="list")

# 干预/引导某个子 Agent
subagents(action="steer", target="agent_id", message="调整方向")

# 终止某个子 Agent
subagents(action="kill", target="agent_id")
```

---

## 📊 协作模式

| 模式 | 描述 | 示例 |
|------|------|------|
| **并行** | 多个 Agent 同时干不同的事 | A 爬数据，B 整理数据，C 生成报告 |
| **串行** | Agent A 干完交给 B | A 收集资料 -> B 分析 -> C 总结 |
| **专家** | 不同 Agent 专注不同领域 | Python专家、爬虫专家、RAG专家 |

---

## 🎯 实际用法

你想让一个任务自动转给别的 Agent 处理，有两种方式：

1. **Spawn 子 Agent** — 我帮你创建一个专门的子 Agent 帮你干活
2. **转发给现有 Agent** — 如果你配置了多个 Agent，可以相互转发
