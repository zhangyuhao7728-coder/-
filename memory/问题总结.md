# 问题总结与修复记录

> 更新日期: 2026-03-15

---

## 1️⃣ 路径问题（已修复）

**原因**：脚本从旧工作目录复制过来时，路径未更新

| 文件 | 错误路径 | 正确路径 |
|------|----------|----------|
| `skill_install_helper.py` | `/Users/zhangyuhao/Learning project/python/...` | `/Users/zhangyuhao/项目/Ai学习系统/...` |
| `git_history_cleaner.py` | 同上 | 同上 |
| `openclaw_auto_maintenance.py` | `~/.nvm/...` 字符串未展开 | 使用变量 `OPENCLAW_CMD` |
| `openclaw_quick_fix.py` | 同上 | 同上 |

**影响**：脚本找不到文件、执行失败、卡住

---

## 2️⃣ Token/上下文爆炸（已防护）

**原因**：
- 无会话消息数量限制
- 无过期会话清理机制
- memory 目录无限增长

**解决方案**：新增 `session_guard.py` 防护脚本

**防护配置**：
```python
MAX_MESSAGES_PER_SESSION = 100  # 单会话最大消息数
MAX_SESSION_AGE_DAYS = 7        # 会话保留天数
MAX_MEMORY_SIZE_MB = 50         # memory 目录最大大小
```

---

## 3️⃣ 潜在风险（已识别待观察）

| 风险 | 说明 |
|------|------|
| 依赖路径硬编码 | 部分脚本用 `~/.nvm/...` 绝对路径，Node 版本升级后会失效 |
| 无日志轮转 | 日志文件可能无限增长 |
| 定时任务缺失 | 清理脚本需手动运行 |

---

## 根因分析

1. **复制粘贴** - 从旧项目复制脚本时未更新路径
2. **缺乏维护** - 脚本写好后没有定期检查和优化
3. **无防护意识** - 没有预防性的清理机制

---

## 维护建议

定期运行以下脚本保持系统健康：

```bash
# 会话防护 (每小时)
python3 scripts/session_guard.py

# 日志清理 (每天)
python3 scripts/log_cleanup.py

# 健康检查 (每周)
python3 openclaw_modules/scripts/health_check.py
```
