# 共享知识库 - 三层懒加载系统

## 核心理念

**按需读取，绝不浪费 token。**

---

## L0 索引层 (启动必读)

**位置**: `knowledge/L0_index.json`

**内容**: 所有知识条目的索引目录

**读取时机**: 每次 Agent 启动时必读

**大小**: < 1KB

```json
{
  "version": "2026-03-02",
  "last_updated": "2026-03-02T19:30:00Z",
  "total_entries": 15,
  "index": [
    {
      "id": "py001",
      "title": "Python 基础语法",
      "tags": ["python", "基础"],
      "layer": "L1",
      "summary_file": "L1/py001_summary.md"
    },
    {
      "id": "ai001",
      "title": "AI 工具使用",
      "tags": ["ai", "chatgpt", "工具"],
      "layer": "L2",
      "full_file": "L2/ai001_full.md"
    }
  ]
}
```

**作用**: 快速知道知识库有什么，不用读全文

---

## L1 摘要层 (需要时读 `knowledge/L1)

**位置**:/*.md`

**内容**: 知识的摘要版本

**读取时机**: L0 索引匹配到相关标签时读取

**大小**: 每条 200-500 字

```markdown
# Python 基础语法 - 摘要

## 核心要点
1. 变量命名规则
2. 缩进语法
3. 常用数据类型

## 快速参考
- 字符串: str
- 数字: int, float
- 列表: list
- 字典: dict

## 记住
Python 用缩进表示代码块！
```

---

## L2 全文层 (确认需要才读)

**位置**: `knowledge/L2/*.md`

**内容**: 完整知识内容

**读取时机**: 明确需要详细信息时才读取

**大小**: 每条 > 2000 字

```markdown
# Python 基础语法 - 全文

## 1. 变量
...

## 2. 数据类型
...

## 3. 控制流
...
```

---

## 使用流程

```
Agent 启动
    ↓
读取 L0_index.json (< 1KB)
    ↓
匹配相关标签
    ↓
需要详情? → 是 → 读取 L1 摘要
    ↓                    ↓
否                  需要深度? → 是 → 读取 L2 全文
                          ↓
                        否 → 结束
```

---

## 共享知识库规则

### 每个 Agent 的 SOUL.md 添加:

```
## 共享知识库

1. 启动时 → 读取 L0_index.json
2. 有相关信息 → 读取 L1 摘要
3. 需要深度 → 读取 L2 全文
4. 学到新知识 → 写入知识库
```

---

## Token 节省效果

| 读取方式 | 传统方式 | 懒加载方式 | 节省 |
|----------|----------|------------|------|
| 启动 | 5KB | 1KB | 80% |
| 查询 | 10KB | 0.5KB | 95% |
| 平均 | - | - | **~80%** |

---

## 维护规则

1. **L0 索引**: 每次更新知识时同步
2. **L1 摘要**: 每周汇总更新
3. **L2 全文**: 按需创建，不过度拆分
4. **自动清理**: 超过 30 天未访问的 L2 转 L1
