## 23:05 上下文管理解决方案

### 问题
上下文爆满但不想丢失记忆

### 解决方案

1. **创建上下文管理脚本**
   - 位置: scripts/上下文管理.py
   - 功能: 定期清理会话，保留关键信息

2. **添加到Cron**
   - 时间: 每周日凌晨3点
   - 自动执行清理

3. **更新AGENTS.md**
   - 添加上下文管理最佳实践
   - 按需读取memory文件
   - 及时保存重要信息

### 核心原则

| 原则 | 说明 |
|------|------|
| 按需读取 | 只读需要的memory文件 |
| 及时保存 | 重要信息立即保存 |
| 定期归档 | 旧会话自动清理 |
| 摘要优先 | 用摘要代替完整记录 |

### 文件大小控制

- 单个memory文件: < 10KB
- 定期清理: 超过7天的会话
- 关键信息: 保存到 MEMORY.md

## 23:26 LLM不保存历史原则

### 核心原则

**LLM不保存历史，只做当前任务推理**

### Memory Layer架构

```
LLM ← Memory Layer
   ↓
  只传: task_summary + current_step + last_result
   ↓
 不传: history + logs + 完整上下文
```

### Memory Layer组成

| 存储 | 用途 |
|------|------|
| vector_db | 向量存储 |
| summaries | 摘要存储 |
| task_state | 任务状态 |
| task_summary | 任务摘要 |

### 禁止事项

- ❌ LLM读取完整history
- ❌ history进入prompt
- ❌ 传递tool_logs给LLM

### 创建的文件

- ~/.openclaw/memory_layer/
  - config.json
  - vector_db/
  - summaries/
- ~/.openclaw/workspace/tools/history_filter.py

✅ LLM不保存历史原则已实施
