你的AI真的安全吗？

实测：OpenClaw安全配置指南

1️⃣ 背景

最近，多家安全机构发布了关于OpenClaw的风险提示。作为用户，我们应该：

  • 理性看待 - 不是不能用，是要正确配置
  • 科学防护 - 了解风险，正确设置
  • 持续关注 - 跟进官方安全更新

本文目的：帮助用户正确配置OpenClaw，降低安全风险

2️⃣ 风险说明（实事求是）

风险说明条件
密钥泄露API密钥被窃取密钥明文写在代码里
权限失控AI执行危险操作未配置权限限制
隐私泄露本地数据被访问未配置文件隔离
公网攻击被黑客扫描入侵Gateway暴露公网

⚠️ 重要：这些风险主要是配置不当导致，正确配置可以大幅降低风险

3️⃣ 配置方案

准备工作：创建一个文件夹存放项目，例如 ~/openclaw

① API密钥保护

问题：密钥明文写在代码里会被他人看到

解决：使用.env文件

cd ~/openclaw touch .env echo 'API_KEY=sk-你的密钥' >> .env

② 用户访问限制

问题：任何人都能访问你的AI

解决：配置白名单

{ "telegram": { "allowFrom": ["你的Telegram ID"] } }

查ID：在Telegram搜索 @userinfobot

③ 命令执行限制

问题：AI可能执行危险命令

解决：命令白名单

{ "security": { "allowedCommands": ["python *", "git *"] } }

④ 文件访问隔离

问题：AI可能访问敏感文件

解决：限定访问目录

{ "security": { "allowedPaths": ["~/openclaw/*"] } }

⑤ 本地绑定

问题：公网暴露被攻击

解决：只绑定本地

{ "gateway": { "bind": "127.0.0.1" } }

说明：127.0.0.1是本地地址，只有本机能访问

⑥ 沙盒模式

作用：将AI运行在隔离环境中

✅ 优点：隔离运行环境、防止权限滥用、保护系统文件

⚠️ 缺点：配置复杂、需要Docker、部分功能受限

配置方法：

  • 方法1：Docker（推荐） - brew install --cask docker
  • 方法2：Podman - brew install podman
  • 方法3：虚拟环境 - python3 -m venv venv
  • 方法4：仅日志 - logCommands: true

💡 选择：新手用方法4，进阶用方法3，生产用方法1

4️⃣ 验证配置

运行检查脚本：

python3 ~/openclaw/tools/安全检查.py

或手动检查：

cat ~/.openclaw/openclaw.json

5️⃣ API获取

  • MiniMax（推荐）：minimax.cn → 注册 → API管理 → 创建密钥
  • 火山引擎：console.volcengineapi.com → 注册 → 子用户 → AccessKey

⚠️ 重要：API密钥只显示一次，请立即保存到.env文件

📝 总结

配置项作用必做
.env文件保护密钥✅ 必须
用户白名单防止滥用✅ 必须
本地绑定防止公网攻击✅ 必须
命令限制防止危险操作✅ 建议
文件隔离保护隐私✅ 建议
沙盒模式隔离运行⭕ 可选
