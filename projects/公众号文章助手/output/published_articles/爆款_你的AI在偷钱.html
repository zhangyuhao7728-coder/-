你的AI真的安全吗？

实测：OpenClaw安全配置指南
📋 实测 🔐 安全 📖 指南

1️⃣ 背景

最近，多家安全机构发布了关于OpenClaw的风险提示。作为用户，我们应该：

  • 理性看待 - 不是不能用，是要正确配置
  • 科学防护 - 了解风险，正确设置
  • 持续关注 - 跟进官方安全更新
本文目的：帮助用户正确配置OpenClaw，降低安全风险

2️⃣ 风险说明（实事求是）

风险 说明 条件
密钥泄露 API密钥被窃取 密钥明文写在代码里
权限失控 AI执行危险操作 未配置权限限制
隐私泄露 本地数据被访问 未配置文件隔离
公网攻击 被黑客扫描入侵 Gateway暴露公网
⚠️ 重要：这些风险主要是配置不当导致，正确配置可以大幅降低风险

3️⃣ 配置方案

📋 准备工作：创建一个文件夹存放项目，例如 ~/openclaw
1

API密钥保护

问题：密钥明文写在代码里会被他人看到

解决：使用.env文件

# 终端执行 # 1. 进入项目目录 cd ~/openclaw # 2. 创建.env文件 touch .env # 3. 写入API密钥（替换成你的） echo 'API_KEY=sk-你的密钥' >> .env
2

用户访问限制

问题：任何人都能访问你的AI

解决：配置白名单

修改 openclaw.json：

{ "telegram": { "allowFrom": [ "你的Telegram ID" ] } }
查ID：在Telegram搜索 @userinfobot，发消息获取
3

命令执行限制

问题：AI可能执行危险命令

解决：命令白名单

{ "security": { "allowedCommands": [ "python *", "git *" ] } }
4

文件访问隔离

问题：AI可能访问敏感文件

解决：限定访问目录

{ "security": { "allowedPaths": [ "~/openclaw/*" ] } }
5

本地绑定

问题：公网暴露被攻击

解决：只绑定本地

{ "gateway": { "bind": "127.0.0.1" } }
说明：127.0.0.1是本地地址，只有本机能访问
6

沙盒模式

作用：将AI运行在隔离环境中

✅ 优点

  • 隔离运行环境
  • 防止权限滥用
  • 保护系统文件
  • 可控制网络访问

⚠️ 缺点

  • 配置复杂
  • 需要Docker
  • 部分功能受限
  • 占用资源

配置方法：

方法1：Docker模式（推荐）

# 1. 安装Docker（macOS） brew install --cask docker # 2. 启动Docker open -a Docker # 3. 配置openclaw.json { "sandbox": { "mode": "docker" } }

方法2：Podman（无需root）

# 安装Podman brew install podman # 启动 podman machine start # 配置 { "sandbox": { "mode": "podman" } }

方法3：简易沙盒（无Docker）

# 使用Python虚拟环境隔离 cd ~/openclaw python3 -m venv venv source venv/bin/activate # 配置限制性环境变量 { "sandbox": { "mode": "venv" } }

方法4：仅日志监控（最简）

# 不使用沙盒，但启用操作日志 { "security": { "logCommands": true, "logFileAccess": true } }
💡 选择建议：
- 新手：方法4（仅日志）
- 进阶：方法3（虚拟环境）
- 生产：方法1（Docker）

4️⃣ 验证配置

运行检查脚本：

python3 ~/openclaw/tools/安全检查.py

或手动检查配置文件 openclaw.json：

cat ~/.openclaw/openclaw.json

5️⃣ API密钥获取

MiniMax（推荐）

  1. 访问 minimax.cn
  2. 注册/登录账号
  3. 进入"API管理"
  4. 创建API Key
  5. 复制保存（只显示一次）

火山引擎

  1. 访问 console.volcengineapi.com
  2. 注册/登录
  3. 创建子用户 → 创建AccessKey
  4. 获取AK/SK
⚠️ 重要：API密钥只显示一次，请立即保存到.env文件

📝 总结

配置项 作用 必做
.env文件 保护密钥 ✅ 必须
用户白名单 防止滥用 ✅ 必须
本地绑定 防止公网攻击 ✅ 必须
命令限制 防止危险操作 ✅ 建议
文件隔离 保护隐私 ✅ 建议
沙盒模式 隔离运行 ⭕ 可选
