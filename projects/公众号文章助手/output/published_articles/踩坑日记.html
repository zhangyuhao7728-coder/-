😅 我的OpenClaw踩坑日记

小白实战总结，看完不再踩坑
👨‍💻 实战 💔 踩坑 ✅ 解决 🔒 安全
📖 前言

作为一名AI小白，我部署OpenClaw的过程简直是一部"踩坑史"...

今天把我的血泪教训分享出来，帮你绕过这些坑！

💔 坑1：配置文件权限泄漏

🔴 错误现象

安全扫描显示：CRITICAL - 配置文件权限过于宽松

OpenClaw拒绝启动，提示权限不安全

✅ 解决方法

# 改成600权限（只有自己可读写） chmod 600 ~/.openclaw/openclaw.json

原因：配置文件里存着API密钥，权限太松会被别人看到！

💡 教训：配置文件权限宁紧勿松！
💔 坑2：Gateway开机不启动

🔴 错误现象

每次开机都要手动启动Gateway

人在外面想用AI，结果连接不上...

✅ 解决方法

三重保护：

# 1. 安装开机启动 openclaw gateway install # 2. 添加到登录项 open AI学习系统.command # 3. 守护进程自动重启 python3 后台守护.py &
💡 教训：配置一次，省心一年！
💔 坑3：API密钥写死在代码里

🔴 错误现象

给朋友演示代码时，

突然发现API密钥暴露了！😱

✅ 解决方法

用.env文件：

# 1. 创建 .env 文件 touch ~/.openclaw/.env # 2. 写入密钥 echo 'API_KEY=sk-xxxx' >> ~/.openclaw/.env # 3. 代码里这样用 from dotenv import load_dotenv load_dotenv() api_key = os.getenv('API_KEY')
💡 教训：密钥绝不进代码！.gitignore也要忽略！
💔 坑4：谁都能用我的AI

🔴 错误现象

突然收到陌生消息

有人在我的AI里问各种问题...

✅ 解决方法

配置白名单：

{ "telegram": { "allowFrom": [ "你的TelegramID" ] } }
💡 教训：只让自己用，别当冤大头！
💔 坑5：AI乱执行危险命令

🔴 错误现象

AI理解错了，

差点删掉重要文件！😰

✅ 解决方法

创建命令白名单：

ALLOWED_COMMANDS = [ "python scripts/写作", "python scripts/分析", "git status", ]

只允许做允许的事

💡 教训：给AI画地为牢！
💔 坑6：网络暴露被攻击

🔴 错误现象

开放了公网访问

收到一堆奇怪登录请求...

✅ 解决方法

只绑定本地：

{ "gateway": { "bind": "127.0.0.1", "port": 18789 } }

只有本机才能访问！

💡 教训：能不放公网就不放！
