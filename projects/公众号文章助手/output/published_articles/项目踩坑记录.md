## ⚙️ 问题一：Telegram连接不稳定

### 🔍 问题现象
- Telegram机器人经常掉线
- 消息发不出去
- 需要手动重启

### ✅ 解决方案

**1. 添加监控脚本**
```python
# ~/.openclaw/scripts/telegram_monitor.py
# 每60秒检查一次，掉线自动重连
```

**2. 开机自启**
```bash
# ~/.zshrc 或 ~/.bash_profile
python3 ~/.openclaw/scripts/telegram_monitor.py &
```

### 💡 经验总结
- 监控脚本要加延迟，避免频繁重连
- 日志要记录，方便排查问题

## ⚙️ 问题二：Gateway总是挂掉

### 🔍 问题现象
- OpenClaw Gateway无响应
- 需要手动重启
- 影响使用

### ✅ 解决方案

**1. 自动监控脚本**
```python
# ~/.openclaw/scripts/gateway_auto_monitor.py
# 2分钟检查一次，挂掉自动重启
```

**2. 连续失败处理**
- 失败3次后进入"积极模式"（30秒检查一次）
- 成功恢复后改回2分钟

**3. Telegram通知**
```python
# 挂掉和恢复时自动发消息通知
```

### 💡 经验总结
- 检查频率要合理：正常2分钟，故障30秒
- 连续失败次数不要设太低，避免误判

## ⚙️ 问题三：API成本失控

### 🔍 问题现象
- 火山引擎API费用暴涨
- 不知道用了多少
- 没有预警

### ✅ 解决方案

**1. 成本监控脚本**
```python
# ~/.openclaw/scripts/cost_controller.py
# 监控使用比例，超过40%自动切换免费模型
```

**2. 预警机制**
| 使用比例 | 动作 |
|----------|------|
| 25% | 提醒 |
| 50% | 警告 |
| 75% | 再次警告 |
| 100% | 强制切换免费模型 |

**3. 自动切换**
```python
if usage > 40%:
   切换到 Ollama 本地模型
```

### 💡 经验总结
- 免费模型（Ollama）要提前配置好
- 切换要有平滑过渡

## ⚙️ 问题四：程序开机不自启

### 🔍 问题现象
- 电脑重启后，所有服务都要手动开
- 麻烦！

### ✅ 解决方案

**1. 使用launchd（Mac）**
```xml
<!-- ~/Library/LaunchAgents/com.ai.monitor.plist -->
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>Label</key>
    <string>com.ai.monitor</string>
    <key>ProgramArguments</key>
    <array>
        <string>python3</string>
        <string>/Users/zhangyuhao/.openclaw/scripts/gateway_auto_monitor.py</string>
    </array>
    <key>RunAtLoad</key>
    <true/>
</dict>
</plist>
```

**2. 开机自动加载**
```bash
launchctl load ~/Library/LaunchAgents/com.ai.monitor.plist
```

### 💡 经验总结
- launchd比cron更可靠
- 要记得 unload 才能删除

## ⚙️ 问题五：Cron任务不执行

### 🔍 问题现象
- 设置了定时任务，但到点没执行
- 日志也没有

### ✅ 解决方案

**1. 检查cron是否运行**
```bash
ps aux | grep cron
```

**2. 查看任务日志**
```bash
log show --predicate 'eventMessage contains "cron"' --last 1h
```

**3. 改用Python内置调度**
```python
# 用schedule库代替cron
import schedule
import time

def job():
    print("执行任务")

schedule.every().day.at("10:00").do(job)

while True:
    schedule.run_pending()
    time.sleep(60)
```

### 💡 经验总结
- Mac的cron有时不太靠谱
- Python的schedule库更灵活

## ⚙️ 问题六：本地模型无法运行

### 🔍 问题现象
- Ollama启动失败
- 模型加载超时
- 显存不足

### ✅ 解决方案

**1. 检查Ollama状态**
```bash
ollama list
ollama serve
```

**2. 只用7B小模型**
```bash
ollama pull qwen2.5:7b
# 别用14B，太大
```

**3. 设置默认模型**
```json
{
  "default": "qwen2.5:7b"
}
```

### 💡 经验总结
- 本地模型要选小的
- 7B足够学习用
